# Scope

## Pages included

This is a single-route, long-scroll proposal site. The top-level app router surface is:

```
/                   Home (full proposal, single scroll)
/audit              Focused view of the Digital Audit & Strategy doc
/proposal           Focused view of the Influencer / PPC / Social proposal
/about              Who prepared this (DSG)
```

Sub-routes for `/audit` and `/proposal` are optional and can be added later if a stakeholder requests a printable or sharable link for a single document.

## Sections included

The home route composes the following sections (in order):

1. **Hero** — Collective positioning, TRUST · ACCESS · COMMUNITY pillars.
2. **Market Context** — ZAR 470M (2024) → ZAR 11B (2028) forecast, CPPA signed 28 May 2024.
3. **Digital Audit Scorecard** — Technical architecture, SEO, mobile, compliance posture.
4. **Competitive Landscape** — Merijane, Merry-Jane.online, Releaf, Cilo Cybin, MG Health, AskMaryJ, Merry2Jane.
5. **Regulatory Reality** — CPPA, SAHPRA, ASA, Google, Meta — what's blocked vs. what's open.
6. **Audience Segments** — Medical patients, wellness seekers, advocates, HCPs.
7. **Channel Strategy** — SEO, Facebook, Instagram, Email, YouTube, LinkedIn, WhatsApp.
8. **Influencer Framework** — Four influencer archetypes + compliance brief template.
9. **PPC Platform Status** — Blocked / Open / Limited status grid + budget framework.
10. **Organic Social Playbook** — Platform-by-platform rules, shadow-ban prevention.
11. **90-Day Roadmap** — Phase 1 Foundations, Phase 2 Build, Phase 3 Scale.
12. **KPIs** — Targets for organic traffic, engagement, list growth, registrations.
13. **Next Steps** — Immediate / Short / Medium term actions.
14. **Compliance Notice** — SAHPRA-aware legal review reminder.

## What is excluded

- **No e-commerce**. Merry-Jane's commercial pathways are out of scope for this site.
- **No member portal integration**. This is a stakeholder pitch site, not a client-facing surface.
- **No live data dashboards**. KPIs are target-based. Live reporting comes post-approval.
- **No multi-language**. English only at launch.
- **No authentication / gating**. A future version may introduce a simple password gate or magic-link share.
- **No CMS**. Content is in-repo (`/src/content`) for speed. A headless CMS is a post-approval consideration.
